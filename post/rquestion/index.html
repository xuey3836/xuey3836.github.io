1. Rtools does not work outside default directory (Windows)

参考, 需要在R中更改Rtool的默认目录。 added the following lines to my global .Rprofile file:

Sys.setenv(PATH = paste("C:/Rtools/bin", Sys.getenv("PATH"), sep=";"))
Sys.setenv(BINPREF = "C:/Rtools/mingw_$(WIN)/bin/"

.Rprofile 文件

在每次R 启动的时候，会加载一些文件，这些文件在以下三个目录，

  • R_home(), R安装的目录；
  • Home, 用户的目录
  • R当前工作目录

为解决上述问题，查看并新建一个.Rprofile文件，

if(!file.exists("~/.Rprofile")) # only create if not already there
  file.create("~/.Rprofile")    # (don't overwrite it)
file.edit("~/.Rprofile")

将自己的Rtools环境添加进去

Sys.setenv(PATH = paste("C:/RBuildTools/3.4/bin", Sys.getenv("PATH"), sep=";"))
Sys.setenv(BINPREF = "C:/RBuildTools/3.4/mingw_64/bin/")

R: tryCatch

how to use tryCatch in R

一个例子

tryCatch 代码

myF <- function(a, b) {
    out <- tryCatch({
        message("Trying.....")
        a/b
    }, error = function(cond) {
        message("You've got an ERROR!")
        message("This is the original message:")
        message(cond)
        return(0)  ## or whatever, e.g. return(NA)
    }, warning = function(cond) {
        message("You'got a WARNING!")
        message("This is the original message:")
        message(cond)
        return(out)
    }, finally = {
        message(paste0("you tried : ", a, "/", b))
    })
    return(out)
}
myF(1, 2)
## Trying.....
## you tried : 1/2
## [1] 0.5
